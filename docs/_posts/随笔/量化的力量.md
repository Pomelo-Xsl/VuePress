> 量化 = 把“感觉”变成“数字”，把“经验”变成“规则”，把“判断”变成“可执行的模型”

## 1、把“感觉”变成“数字”
### 例子一：健身（我最近练得怎么样，要不要加重量）
**感觉阶段**
> “最近练得还行，好像有点进步。”

**数字阶段（量化感觉）**
> “这周卧推从 70kg × 5 次,提升到 75kg × 5 次”

## 2、把“经验”变成“规则”
### 例子一：健身（我最近练得怎么样，要不要加重量）
**原始经验（不可执行）**
> 如果我连续几次都能轻松完成，就可以加重量

**规则化经验**
```text
如果：
- 连续 3 次训练
- 每次都完成 5×5
- 最后一组主观强度 RPE ≤ 7

那么：
- 下次加重量 2.5kg
```

## 3、把“判断”变成“可执行的模型”
### 例子一：健身（我最近练得怎么样，要不要加重量）

> 但现实更复杂，你心里其实在综合判断：“要不要加重量，还得看恢复、睡眠、状态。”

**人脑判断（不可复现）**
>  “今天状态一般，还是别加了”

**模型化判断**
```text
输入指标：
- 上次训练完成度
- 最近 3 天睡眠时长
- 心率恢复
- 训练主观疲劳度

评分模型：
训练状态分 =
  完成度 × 0.4
+ 睡眠分 × 0.3
+ 疲劳分 × 0.3

输出：
- ≥ 80 分 → 加重量
- 60~80 → 维持
- < 60 → 降重量
```

## 4、总结
**以后做什么事情都要量化，比如：健身小白板块、理财入门板块、自媒体浅尝板块、计算机知识体系的建立板块等**
